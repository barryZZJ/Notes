If you'd prefer that conda's base environment not be activated on startup, 
   set the auto_activate_base parameter to false: 

conda config --set auto_activate_base false

在bat中初始化conda环境：
win:
''call activate.bat''
''call conda activate xxx''

linux:
''eval "$(/root/miniconda3/bin/conda shell.bash hook)"''

conda create -n 名字 python==3.x --clone exist_name -y

conda env remove -n 名字

conda info -e

环境迁移：
需要同一平台同一系统
1. 直接拷贝env下的文件夹到目标文件系统
2. 使用 conda-pack
	a. conda install conda-pack
	b. conda pack -n <env> 生成.tar.gz
	c. 拷贝到目标系统的env下
	d. tar -zxvf <file>.tar.gz
