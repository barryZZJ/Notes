环境迁移：
需要同一平台同一系统
1. 直接拷贝env下的文件夹到目标文件系统
2. 使用 conda-pack
	a. conda install conda-pack -y
	b. conda pack -n <env> 生成.tar.gz  (--ignore-missing-files)
	c. 拷贝到目标系统的env下
	d. mkdir <env> && tar -zxvf <env>.tar.gz -C <env>
	e. conda activate <env>
	f. conda-unpack (每次更换路径后都需要运行)

换源
conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
conda config --set show_channel_urls yes
conda config --show channels

conda config --remove channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/

conda config --add channels defaults
conda config --add channels conda-forge
~/.condarc中调整顺序，给与conda-forge最高优先级

conda-forge 镜像:
conda config --set custom_channels.conda-forge https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/

安装库时指定channel：
-c conda-forge
-c https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/conda-forge/
可以指定多个channel，优先级从左到右
conda config --set channel_priority strict 严格保证优先级顺序 默认为flexible

导出依赖
conda env export > environment.yml
安装新环境
conda env create -f environment.yml[ -n name]
更新当前环境（删除yml中的name，或带-n参数）
conda env update -f environment.yml -n <env>
conda install --file requirements.txt

install:
--freeze-installed, --no-update-deps Do not update or change already-installed dependencies. （经测试不是绝对的，参考下面的package pinning）
pin packages： https://docs.conda.io/projects/conda/en/latest/user-guide/tasks/manage-pkgs.html#preventing-packages-from-updating-pinning
-d, --dry-run
--no-deps 不改变依赖

环境泄露，进不到虚拟环境里：
检查:
~/.bashrc
~/.bash_profile
里有没有export PATH指令的干扰，需要移动到~/.bashrc的conda initilize之前，然后重启终端即可。
