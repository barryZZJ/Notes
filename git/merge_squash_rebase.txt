squash 是把所有commit合并成一个commit，可以得到干净的历史记录
In squash mode, changes are combined into a single commit on master branch, looks quite neat.
details are gone with the feature branch

* Run the command git rebase -i HEAD~n, where "n" is the number of latest commits you want to squash. This will open a text editor with a list of the commits you want to squash.
* Change the word "pick" to "squash" for all the commits except the first one. This tells Git to squash those commits into the first commit.
* Save and close the text editor. Git will then automatically rebase your changes and prompt you to enter a new commit message for the squashed commit.
* Edit the commit message to accurately reflect the changes that were made in the squashed commits.
* Save and close the text editor again. Git will then create a new, squashed commit with the changes from all the previous commits.

rebase 是把当前分支的commit作为patch依次打在main上（把当前分支的commit重放一遍），不会合并。使用interactive rebase会自动合并fixup, amend之类的提交。
In rebase mode, all commits are added onto master branch individually, a little verbose.
details are retained at the cost of “verboseness”.
选rebase current onto selected，把当前分支的修改依次打在selected上

merge 不能保持 master 分支干净，保持了所有的 commit history，
In merge mode, an ugly sideway out-and-in retains all details in developing the feature.

pull时使用rebase：
git pull --rebase
pull时默认使用rebase：
git config --global pull.rebase true
