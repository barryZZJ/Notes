添加：
git submodule add [-b <branch>] 仓库链接 [path]

会生成一个.gitmodules文件：
[submodule "cqu_auth"]
	path = cqu_auth
	url = git@github.com:barryZZJ/CQU_Authenticator.git

拉取上游更新：
git submodule update --remote --merge --recursive
变基到上游更新：
git submodule update --remote --rebase

本地改动后推送：
手动进入子模块推送
或 git push --recurse-submodules=on-demand
可以通过设置 git config push.recurseSubmodules on-demand 让它成为默认行为。

克隆项目
git clone --recurse-submodules
或：
git submodule update --init --recursive

查看：
git submodule

移动
git mv source dest

删除：
git rm <path-to-submodule>
rm -rf .git/modules/<path-to-submodule>
git config --remove-section submodule.<path-to-submodule>.

子模块要么用相对引用，要么就必须手动在__init__里添加路径到sys.path
