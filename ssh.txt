=== ssh服务器： ===
Linux:
1. 安装：sudo apt install openssh-server
2. chmod 700 .ssh
3. 把id_rsa.pub里的内容写入~/.ssh/authorized_keys，并chmod 600 或 644
4. sudo nano **/etc/ssh/sshd_config**文件，设置（默认是yes，不需要动，也不用重启ssh服务）
PubkeyAuthentication yes
AuthorizedKeysFile .ssh/authorized_keys
5. systemctl daemon-reload
6. systemctl restart ssh.socket 或 systemctl restart sshd

windows ssh 服务器: OpenSSH
1. 设置—系统—可选功能——”添加可选功能“搜索OpenSSH 服务器，安装
2. 安装后服务名：**sshd** (OpenSSH SSH Server)
3. 配置：
https://winscp.net/eng/docs/guide_windows_openssh_server
windows条目说明：https://learn.microsoft.com/en-us/windows-server/administration/openssh/openssh_server_configuration
linux条目说明：https://man.openbsd.org/sshd_config
https://github.com/PowerShell/Win32-OpenSSH/wiki/sshd_config
配置文件：C:\ProgramData\ssh\sshd_config
Port 23355
ListenAddress 0.0.0.0
ListenAddress :: #同时添加这一行，以支持ipv6
#连接后默认文件夹为/C:/Users/<username>，要想修改，添加-d参数：
# (https://man.openbsd.org/cgi-bin/man.cgi/OpenBSD-current/man8/sftp-server.8?query=sftp-server&sec=8
Subsystem   sftp    sftp-server.exe -d "D:"
# 只启动sftp（？）
ForceCommand internal-sftp
4. 配置administrators_authorized_keys权限（见下）
5. 配置ssh文件夹权限（同上）
6. 防火墙添加入站规则（注意修改端口号）：
管理员Powershell：''New-NetFirewallRule -Name sshd -DisplayName 'OpenSSH Server (sshd)' -Enabled True -Direction Inbound -Protocol TCP -Action Allow -LocalPort 22''
或手动添加：
OpenSSH SSH Server (sshd)
程序：%SystemRoot%\system32\OpenSSH\sshd.exe
协议：TCP
端口：23355
6. 启动sshd服务并设为开机自动启动：服务(sshd)启动类型设为自动

**其他配置选项说明：**
Log:
LogLevel INFO | DEBUG | DEBUG3
日志文件位置 SyslogFacility项指定：
默认 AUTH，日志在事件查看器-应用程序和服务日志-OpenSSH
设为 LOCAL0 则日志在%programdata%\ssh\logs

Linux下AUTH对应/var/log/auth.log（如 Ubuntu、Debian 等基于 Debian 的系统）
/var/log/secure（如 CentOS、Red Hat、Fedora 等基于 Red Hat 的系统）

DEBUG:
关闭sshd服务，自己在**管理员**命令行运行sshd -d 或 sshd -ddd

**配置密钥登录：**https://learn.microsoft.com/en-us/windows-server/administration/openssh/openssh_keymanagement#deploying-the-public-key
（电脑名不要和用户名相同，否则会bug）
# 禁止密码登录，保证安全性
PasswordAuthentication no
# 开启密钥文件登录
PubkeyAuthentication yes
# 设置白名单（不在的不能连，其他的如AllowGroups, DenyUsers https://learn.microsoft.com/en-us/windows-server/administration/openssh/openssh_server_configuration#allowgroups-allowusers-denygroups-denyusers%EF%BC%89
AllowUsers barry sshadmin
# 针对不同用户的不同设置可以使用Match https://learn.microsoft.com/en-us/windows-server/administration/openssh/openssh_server_configuration#match
Match User sshadmin
	PasswordAuthentication yes
#设置密钥文件存储位置
''AuthorizedKeysFile	.ssh/authorized_keys''
#并保留（在最下面）：
''Match Group administrators''
	  ''AuthorizedKeysFile __PROGRAMDATA__/ssh/administrators_authorized_keys''

**添加客户端密钥：**https://learn.microsoft.com/en-us/windows-server/administration/openssh/openssh_server_configuration#authorizedkeysfile
1. 客户端生成密钥 ssh-keygen -t rsa
2. 服务端新建 AuthorizedKeysFile 对应的文件：（即时生效）
【windows】
【登录的是普通用户】''.ssh/authorized_keys，''相对于%userprofile%（C:\用户\<username>），想登录哪个用户就在哪个目录下配置：
2.1 新建.ssh文件夹（可以酌情调一下权限，经测试不影响）
2.2 在.ssh内新建authorized_keys文件**（注意权限和所有者！）**：
高级权限：取消继承、**只有System和该用户**完全控制（如果有其他用户则会导致Authentication Refused），**所有者必须也为该用户**，才可密钥登陆。
【登陆的是管理员用户】路径为%programdata%\ssh\administrators_authorized_keys
2.1 高级权限：取消继承、**只有System和Administrators组**完全控制，所有者必须为**SYSTEM**。
	管理员命令行设置文件权限：icacls.exe "%programdata%\ssh\administrators_authorized_keys" /inheritance:r /grant "Administrators:F" /grant "SYSTEM:F"
注意每次更改完需要重新调整ssh文件夹的权限，否则会报错不让连接
【linux】在~/.ssh/authorized_keys下做相同操作，chmod 600即可，即时生效。

在新建的 AuthorizedKeysFile 中写入客户端生成的公钥(.pub)（若权限不足，用管理员打开vscode编辑；或临时修改所有者；或以对应用户登录再修改）（更新后不用重启sshd服务）
windows：手动放入
linux：ssh-copy-id user@host -p <port>

（Windows不可用：sshd_config中的 PermitRootLogin prohibit-password改为 yes）
