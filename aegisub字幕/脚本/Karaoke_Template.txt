===== Effect =====
https://aeg-dev.github.io/AegiSite/docs/3.2/automation/karaoke_templater/template_modifiers/
template <modifier=syl>: 模板行
code <modifier=once>: 代码行（如果是定义函数/全局变量使用code once，如果要应用于每一行/每一个音节使用code line/syl）（all应用于所有样式）
空白、且样式相同: timed karaoke line
fx: styled karaoke line
如果没有k轴，则整行认为是一个音节

代码行修改台词内容：
<Effect>code once, <Text>function f() nt=""; for c in _G.unicode.chars(orgline.text) do nt=nt..string.format("{\\t(%f,0,%s}", 0.1,"1")..c; end return nt; end
<Effect>template line notext, <Text>!f()!

xres, yres = _G.aegisub.video_size() if xres == nil  then x_rat = 1 y_rat = 1 else x_rat = xres/720 y_rat = x_rat end if (xres and xres ~= meta.res_x) or (yres and yres ~= meta.res_y) then _G.aegisub.debug.out(2, "Warning: script resolution %dx%d does not match with loaded video resolution %dx%d resample script resolution", meta.res_x, meta.res_y, xres, yres) end

msa = _G.aegisub.ms_from_frame(1) msb = _G.aegisub.ms_from_frame(100) if msa == nil and msb == nil then _G.aegisub.debug.out(2, "Warning: Please load the video or use dummy video.Frame duration was provisionally set to 33 milliseconds. ") frame_dur = 33  else frame_dur = (msb-msa)/100 end
计算一行的帧个数：frame_dur / <line_dur> == frame_num

====================
code line all once
template pre-line name all
template line name notext all
template syl noblank
**Note: 如果template syl运行后没反应是因为没有k轴！可以改用template char**
====================
once: 只用于代码行，定义函数
template行：对每个syl执行一次，只生成一行；code行：对每行执行一次；template行可以包括name，name相同的会按顺序合并在一起(可以不是k轴)
line [name]: 
	如 line effect
	如果只想每行执行一次的话使用pre-line
pre-line [name]: 只用于template行，在[name]相同的line模板输出结果前添加(只执行一次)
	如果没有对应的line行则默认line行为karaoke行的原内容
	要想实现notext的效果，需要手动加一个空白的同名line行，且标注notext
syl: 对每个音节执行（每个\k之间是一个音节），每个音节生成一行(**必须是k轴**，否则相当于line。加noblank去掉空行)
syl char: 对每个character而不是每个syllable执行(不用是k轴，仍作用于每个字符)
furi: 作用于振假名/注音假名
all: 对所有样式都执行
fx <name>: 对inline-fx为<name>的syllable进行额外处理
fxgroup <name>
keeptags: 不支持与char或multi连用
multi: 作用于multi-highlight timed karaoke的每个highlight
noblank: 模板不会作用于blank(/空白)字符。（一般都开启）
	A syllable is considered blank if its tag-stripped text consists only of a combination of ASCII whitespace characters and ideographic fullwidth space characters, or is completely empty. A syllable is also considered empty if its duration is zero.
notext: 不保留字幕文本。用于输出矢量绘图。【不可用于code行】
repeat <n>或loop <n>: 模板会(对每个syllable)重复执行n次，n必须是常数。循环变量用j, maxj或tenv.j, tenv.maxj获取

定义全局变量（如果是随机数则每行都不一样）：
<Effect> code line all <Text> var=0;
<Effect> code line all <Text> function f(arg) var=var+arg; return ""; end
（如果需要存储随机数可以用remember() recall())

=== Utility functions ===
**!retime!**
https://aegisub.org/docs/latest/automation/karaoke_templater/code_execution_environment/#retime
生成新时间为原时间的什么位置。
It adjusts the start and end time of the output line in various ways.
retime(mode, startadjust, endadjust)
Possible _mode_s:
* abs or set - Both startadjust and endadjust are used as absolute time values to set the start and end time of the line directly.
* preline (-x, 0) - Intended to make effects that happen before the actual line start. Both start and end time of the line are set to the start time of the line, then startadjust is added to the start time and endadjust added to the end time. Usually startadjust should be negative here and endadjust be zero.
* line - Use the regular line timings and just add startadjust to the start time and endadjust to the end time.
* start2syl - Intended to make the look of the syllable from the start of the line until it is highlighted. The start time of the line is kept and the end time is set to the start time of the syllable. Use startadjust and endadjust to offset the times.
* presyl (-x, 0)- Similar to preline but for the syllable timing instead.
* syl - From start of syllable to end of syllable.
* postsyl (0, x)- Similar to presyl but the base timing is the syllable end time instead of start time. You will usually want to use a zero addstart and positive addend here.
* syl2end - The time from the end of the syllable to the end of line, similar to start2syl.
* postline (0, x)- Similar to postsyl but for the line timing instead. 生成结果的起始时间=orgline.end+startadjust，结束时间=orgline.end+endadjust
$lstart, $lend, $ldur等与时间相关的var均为retime之前的时间

===== Inline Variables ($-variables) =====
https://aeg-dev.github.io/AegiSite/docs/3.2/automation/karaoke_templater/inline_variables/
使用$var引用，如需运算需要两端加上!!，如!$var!
!!之间当作lua程序处理
【只能在template行使用，不能在code行使用】


包含的var如下：
**Automatic variants**
In pre-line templates the automatic inline variables refer to the line variants, and everywhere else they refer to the syllable variants.

start, end, mid
	start and and midway time for line/syllable; absolute for lines and relative for syllables
dur, kdur
	duration in milliseconds and centiseconds of line/syllable
i
	line or syllable index
left, center, right
	left, center and right edges of line/syllable, absolute from left screen edge
top, middle, bottom
	top middle and bottom edges of line/syllable, absolute from top screen edge
x, y
	x and y position of line/syllable when using default alignment
width, height
	width and height of line/syllable in pixels, this is rounded and might not match exactly with the positioning variables

**Line variants**
【这里获取到的都是orgline的，要想获得修改后的，使用!line.start_time!,!line.duration!等】【但是line.left等位置不能获取被\pos修改后的，而且就算在样式里改frz也获取不到】
【位置相关的参数都是整数，要想获取小数，使用!line.width!等line对象的属性】
layer
	line layer
lstart, lend, ldur, lmid
	line start time, end time, duration and midway, all absolute times in milliseconds
style
	name of the line style
actor
	name of the line actor
margin_l, margin_r
	effective left and right margin (line if nonzero, otherwise style)
margin_v, margin_t, margin_b
	effective vertical, top and bottom margin, vertical and top is same
syln
	number of syllables on line
li
	line index (first physical line in file is 1)
lleft, lcenter, lright (int) 【lleft和lright能保证在实际值的外面，不会截断】
	line left, horizontal center and right edges, taking margins and alignment into account, rounded to an integer value
ltop, lmiddle, lbottom
	line top, vertical middle and bottom edges, taking margins and alignment into account, rounded
lx, ly
	line x and y position suitable for a \pos command when alignment is not overridden
lwidth, lheight
	line width and height in pixels, this is rounded and might not match exactly with the positioning variables

**Syllable variants【使用syl获取修改后的？，参考上面的line】**
sstart, send, smid
	syllable start, end and midway times relative to start of line, suitable for putting into \t and \move
sdur, skdur
	syllable duration in milliseconds and centiseconds
si
	syllable index from start of line
sleft, scenter, sright (int)
	**absolute** left, horizontal center and right edges for syllable from left edge of screen, suitable directly for \pos and \move
sbottom, smiddle, stop
	absolute bottom, vertical middle and top edges for syllable from top edge of screen, suitable directly for \pos and \move, adjusted for furigana positioning if needed
sx, sy
	syllable absolute x and y position in default alignment, suitable for using directly in \pos and \move
swidth, sheight
	syllable width and height in pixelsthis is rounded and might not match exactly with the positioning variables
