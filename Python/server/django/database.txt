settings.py中配置数据库
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': os.path.join(BASE_DIR, 'db.sqlite3'),
    }
}
ENGINE: 数据库类型 # mysql: django.db.backends.mysql
NAME: 数据库的物理存储地址

在models.py里设置ORM（对象关系模型）的类对象。
1. 必须继承models.Model类
2. 类名即自动生成的数据表名称的一部分
3. 类中的成员属性即表的字段名称
属性：
primary_key=True
max_length=20
unique=True
on_delete=models.CASCADE/DO_NOTHING

class TableName(models.Model):
    id = models.AutoField(primary_key=True)
    name = models.CharField(max_length=20) # varchar(20)

自增类型 AutoField
可变字符串类型 CharField
日期类型 DateTimeField
整型 IntegerField
小数 FloatField

外键 ForeignKey(表名， on_delete=models.CASCADE/DO_NOTHING)
一对一映射 OneToOneField(表名, unique=True)
    

生成/更新数据库：
python manage.py makemigrations
python manage.py migrate
------------------------------------
python3 manage.py makemigrations app名  # 让 Django 知道我们在我们的模型有一些变更
python3 manage.py migrate app名   # 创建表结构

Django 会自动添加一个 id 作为主键。

from app.models import Model

添加：
# insert into Model表 values(值1, ...)
obj = Model(成员变量1=值1, ...)
obj.save()
或
Model.objects.create(成员变量1=值1, ...)

查询：
# select * from Model表
items = Model.objects.all() # type: QuerySet
# 返回的是多条记录构成的表

# 输出所有数据
    response1 = ''
    for item in items:
        response1 += item.字段名 + " "
    response = response1
    return HttpResponse("<p>" + response + "</p>")

# WHERE id=1 （根据条件不同可以返回多个）
response = Model表.objects.filter(id=1) # 成员变量=值

# 获取单个对象
response = Model表.objects.get(id=1) # 成员变量=值

# 限制返回的数据 相当于 SQL 中的 OFFSET 0 LIMIT 2;
Test.objects.order_by('name')[0:2]

# 数据排序
Test.objects.order_by("id")

# 上面的方法可以连锁使用
Test.objects.filter(name="runoob").order_by("id")

更新：
# UPDATE Model SET 字段1=新值1... WHERE 字段=值
Model.objects.filter(字段=值).update(字段1=新值1, ...)
或
obj = Model.objects.filter(...)
obj.字段 = 新值
obj.save()

删除：
# DELETE FROM Model WHERE id=1
obj = Model.objects.filter(id=1)
obj.delete()
或
Model.objects.filter(id=1).delete()
