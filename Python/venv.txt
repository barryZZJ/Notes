1.自带虚拟环境venv（√）它的很多操作都和 virtualenv 类似，但是两者运行机制不同
https://docs.python.org/zh-cn/3/tutorial/venv.html

新建：
python3 -m venv venv_path
<or> virtualenv venv_path
不能指定版本，与当前版本相同

# windows:
venv\Scripts\activate.bat
（每次os.system都会启动一个新的shell，因此要想通过python启动需要用subprocess或cmd /c "..."一行字符串完成）

bat中启动虚拟环境：
call venv\Scripts\activate.bat

# linux 控制台/sh中:
source venv/bin/activate # (完整路径，~要改成/<user>)
deactivate

路径不能包含中文，否则pycharm无法在终端自动激活虚拟环境

使用别人的虚拟环境时需要本地安装相同版本的python，
然后更新虚拟环境中的系统python路径：
python -m venv --upgrade path_to_venv_root

修好python之后手动修pip：
python -m pip install --upgrade pip 或 python -m ensurepip --default-pip

修改宿主python的环境：pyvenv.cfg

克隆虚拟环境：
https://github.com/edwardgeorge/virtualenv-clone
pip install virtualenv-clone
virtualenv-clone venv1/ venv2/
OR
pip freeze > requirements.txt
pip install -r requirements.txt
自动化：
{{{code: lang="python3" linenumbers="False"
import os
from os import path

src_base = r"C:\Coding2\PyProjects\venv"
srcs=[
    "zmsub", "aegisub", "bot", "bypy", "dp", "ptbcontrib", "publish", "tgbot", "webdev", "wecomsan"
]

dst_base = r'E:\Coding2\PyProjects\venv'

def move(src, dst):
    if path.exists(dst):
        print('skip', dst)
        return
    print('moving', dst)
    req = dst + r'\req.txt'
    os.system(
        'cmd /c "'
        'python3 -m venv ' + dst + ' && '
        'call ' + src + r'\Scripts\activate.bat' + ' && '
        'where python' + ' && '
        'pip freeze > ' + req + ' && '
        'deactivate' + '"'
    )
    print()
    os.system(
        'cmd /c "'
        'call ' + dst + r'\Scripts\activate.bat' + ' && '
        'where python' + ' && '
        'python -m pip install --upgrade pip' + ' && '
        'pip install -r ' + req + ' && '
        'deactivate' + '"'
    )
    print()
    print('done!')

for src in srcs:
    move(src_base + "\\" + src, dst_base + "\\" + src)
}}}
