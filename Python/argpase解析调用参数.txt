argparse.ArgumentParser(description='A foo that bars')
PS：不带参数运行时不会显示，因此代码需要判断，如果没有参数，则调用parser.print_help()然后退出
说明信息设为多行(使用RawDescriptionHelpFormatter为formatter_class参数)
	https://docs.python.org/3/library/argparse.html#formatter-class
	https://stackoverflow.com/questions/3853722/how-to-insert-newlines-on-argparse-help-text

# 位置参数（必须参数）
parser.add_argumdeent("some", help="helpful text", type=int)
add_argument参数：
nargs:  '+' 可获取多个参数，返回一个列表；整数：读取指定个数的参数
help 帮助说明
type 保存类型
default 默认值
metavar
choices

# 可选参数（flag）
parser.add_argument("-v", "--verbose", action="store_true") # 使用-v时该选项为true，否则为false
# 接收参数
parser.add_argument("-m", "--msg", type=int, default=1, choices=[0,1,2])
parser.add_argument("-v", type=bool, default=False) # 这种情况下不带-v则为False，只要带了，就算-v False也是True

metavar="xxx" : 改变帮助内容里显示的参数名

# 解析程序参数
args = parser.parse_args()
# 解析自定义参数
args = parser.parse_args(['arg1', '-o', 'arg2'])

获取参数：
args.some (如果add_argument中为'-'，这里需要使用'_'；大小写保持不变)
args.verbose
args.msg

存储：
vars(args)，输出字典

手动处理参数错误的情况（可以用于exe双击执行时自定义处理方式）(python>=3.9)
1. https://docs.python.org/3.9/library/argparse.html#exit-on-error
argparse.ArgumentParser(exit_on_error=False)
catch argparse.ArgumentError
2. https://docs.python.org/3.9/library/argparse.html#exiting-methods
修改parser.error
3. 继承并重载parser.error()函数

注意不会包含两边的双引号（会包含两边的单引号）

{{{code: lang="python3" linenumbers="False"
import argparse

def my_error(message):
    print("error:", message, file=sys.stderr)
    sys.exit(1)

parser = argparse.ArgumentParser(description='Process some integers.')
parser.add_argument('integers', metavar='N', type=int, nargs='+',
                    help='an integer to be processed')
parser.error = my_error

args = parser.parse_args()
print(args.integers)

}}}
