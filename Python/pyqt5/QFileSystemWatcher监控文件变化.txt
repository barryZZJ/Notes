====== QFileSystemWatcher监控文件变化 ======
Created 星期四 08 十二月 2022

https://doc.qt.io/qt-6/qfilesystemwatcher.html

.addPath()  -> bool 添加成功返回True，已经添加过或添加失败返回False
.addPaths()  -> list[str] 失败的路径
.removePath()  -> bool  文件夹不存在时会返回False
.removePaths()  -> list

监控指定文件路径的添加、删除、修改：
.files() 查看正在监控的文件
fileChanged  signal  文件发生修改时，返回该文件的路径

监控指定文件夹内容(子文件夹无法监控)的增加、修改、删除：
.directories() 查看正在监控的目录  存储既可以有绝对路径也可以有相对路径
directoryChanged  signal 文件夹内容发生变化（文件增删）时，返回被监视的文件夹路径

注意：大文件拷贝时，信号是在拷贝开始时产生的
临时解决办法检查拷贝是否完成：
	首先检查是否path.is_file()
	不断检查文件上次修改时间，超过1s后认为复制完成；（注意使用RDP复制的文件只有开始复制和完成复制的时候修改时间才会变）
	x不断检查文件大小，一定时间内大小稳定则认为复制完成；（不行，文件大小是一开始就分配了的）
	或尝试rb+模式打开，如果不报PermissionError，则拷贝完成，如果不存在报FileNotFoundError（注意不能检测到暂停复制的情况。（注意使用RDP复制的文件会一直有访问权限）

用法： https://blog.csdn.net/liang19890820/article/details/51849252
监视文件夹：
建立一个字典，按路径分类，currFiles[path]=set(被监视目录下的文件)，用于获取具体内容的变动情况
directoryUpdated:
某个路径发生变动时，该目录文件转换为集合，使用集合的差运算，
old-new = deleted, new-old=added
len(deleted)==len(added)==1为文件重命名，其他情况为文件增、删
