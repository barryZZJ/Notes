====== Pyinstaller ======

（在macOS中运行则会生成mac程序）

打包成exe：
https://pyinstaller.org/en/stable/index.html
pip install pyinstaller

cmd中自动化：
@echo off
call "E:\Coding2\PyProjects\venv\zmsub\Scripts\activate.bat"
pyinstaller compresser.py -y --clean --add-data src;src -F --clean -n name --icon=xx.ico

如果报错，记得-p添加site-packages目录, e.g. xxx\envs\script\Lib\site-packages 
可发布文件在dist文件夹中

如果依赖外部文件，需要用--add-data添加

打包时报“无法成功完成操作，因为文件包含病毒或潜在的垃圾软件。”，需要把实时防护关掉

=== pyinstaller>4.2路径问题： ===
打包成单个文件后，python环境/附加文件会在tmp文件夹下，__file__返回的也是这个路径下的py文件(但文件不存在)
exe文件路径：''os.path.dirname(sys.executable)'' 双击运行时，os.getcwd()返回exe所在目录
--runtime-tmpdir PATH指定tmp文件夹
os版：
''import os''
''import sys''

''ISEXE = hasattr(sys, 'frozen')''
''BASE = os.path.dirname(sys.executable if ISEXE else __file__)  # 打包后的exe所在目录。打包前为该py文件目录。''
''BASE_TMP = sys._MEIPASS if ISEXE else BASE # 用于读取附加文件。打包后运行时，如果-F单文件，则为系统临时文件所在路径（解包到临时文件夹）；如果是-D目录，则是打包后exe所在路径\_internal（add-data的默认起始路径，--contents-directory可以修改）。打包前为该py所在目录。''


Path版：
''import os''
''import sys''
''from pathlib import Path''

''ISEXE = hasattr(sys, 'frozen')''
''BASE = Path(os.path.dirname(sys.executable if ISEXE else os.path.dirname(__file__)))''
''BASE_TMP = Path(sys._MEIPASS) if ISEXE else BASE''

注意：python>=3.9不再支持win7

参数
-p DIR，--path=DIR	设置 Python import模块的路径（和设置 PYTHONPATH 环境变量的作用相似）。也可使用路径分隔符（Windows 使用分号，Linux 使用冒号）来分隔多个路径
-n NAME，--name=NAME	指定项目（产生的 spec）名字。如果省略该选项，那么第一个脚本的主文件名将作为 spec 的名字
--clean 构造成功后清理缓存文件
-y, --noconfirm 不提示是否覆盖
-F，--onefile	产生单个的可执行文件
-D，--onedir	产生一个目录（包含多个文件）作为可执行程序 （默认）
-a，--ascii	不包含 Unicode 字符集支持
-d，--debug	产生 debug 版本的可执行文件
-w，--windowed，--noconsolc	运行时不显示命令行窗口（仅对 Windows 有效）
-c，--nowindowed，--console	运行时显示命令行窗口（仅对 Windows 有效） （默认）
--log-level LEVEL   TRACE, DEBUG, INFO, WARN, ERROR, CRITICAL.  default: INFO
--add-data <SRC;DEST or SRC:DEST> Additional non-binary files or folders to be added to the executable. The path separator is platform specific, os.pathsep (which is ; on Windows and : on most unix systems) is used. This option can be used multiple times. 附加其他文件, DEST可以为.或\表示根目录，访问路径为sys._MEIPASS（系统tmp文件夹(-F)/exe文件夹(-D)）支持直接写文件夹，但加了*会把里面的文件放到同一级目录
--add-binary <SRC;DEST or SRC:DEST> Additional binary files to be added to the executable. 附加其他exe文件, DEST可以为.或\表示根目录，访问路径为sys._MEIPASS（系统tmp文件夹）
-r --resource FILE[,TYPE[,NAME[,LANGUAGE]]]
-i --icon FILE.ico
 --disable-windowed-traceback
--hiddenimport MODULENAME 有些hidden import的库需要手动import进来，如--hidden-import=torch --hidden-import=torch.utils.bacqkends --hidden-import=torchvision 查看所有hidden [[import%E7%9A%84%E5%BA%93%E7%9A%84%E6%96%B9%E6%B3%95https://pyinstaller.org/en/stable/when-things-go-wrong.html#listing-hidden-imports]]
--contents-directory . # -D 专用，指定add-data加到哪，“.”表示与exe在同一路径

pyinstaller --clean 清理缓存


输出目录设置：
--distpath DIR   default: ./dist
--workpath WORKPATH    default: ./build
--specpath DIR    default: ./

spec文件：
设置中文：
name=u'中文名'

报错解决file already exists but should not:
打包pyqt5时可能会弹出warning，
在xx.spec文件里a=Analysis()后添加：
for b in a.binaries:
	txt = '_C.cp38-win_amd64.pyd'
	if txt in b[1] or txt in b[0]:
		print(f"Removing {b}")
		a.binaries.remove(b)
		break
后，使用pyinstaller xx.spec重新编译（不需要任何额外参数）

使用InstallerForge工具构建程序的安装文件。

=== 判断是否是打包后的可执行文件： ===
hasattr(sys, 'frozen')
或
getattr(sys, 'frozen', False)

=== Splash ===
--splash IMAGE_FILE

=== 生成多个exe ===
https://www.zacoding.com/en/post/pyinstaller-create-multiple-executables/
使用pyi-makespec分别生成各自的spec，然后整合在一起共用一个COLLECT函数 (参数与pyinstaller一致）

pyinstaller PublishAssistant.py -w -y --clean --add-data README.md;./ --add-data bin/;./bin/ --add-data icon.ico;./ -i icon.ico
}}}
