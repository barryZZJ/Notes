对于CPU-bound的任务，使用Python多线程并**不会**显著提高CPU利用率，因为全局解释器锁（Global Interpreter Lock，GIL）的存在，要想提高效率应考虑多进程。
多线程只会提升IO-bound的任务的效率，因为可以解决IO等待时间

from threading import Thread, Lock
t = Thread(target=func, args=[args])
t.start()

1. 继承threading.Thread
class myThread(Thread):
	def __init__(self, args):
		super().__init__()
		self.args = args
		self.lock = Lock()

	def run(self): # 线程在创建后会直接运行run函数
		self.lock.aquire() # 加锁
		# ...
		self.lock.release() # 解锁
		pass
myThread(args).start()

默认多线程时无法响应Ctrl+C，因为只有主线程停止，而子线程没有停止。
解决办法：
1. 各个子线程设置setDaemon(True)，主线程最后加一个while True循环，主线程响应退出信号时顺带销毁子线程（强杀）。


守护线程 Daemon thread：
子线程.setDaemon(True) 代表让子线程跟随主线程销毁
必须在start()方法调用之前设置
设置后主线程（进程）结束了，不管子线程是否完成，必须也跟着结束。不设置的话，主线程结束后，子线程也还在执行。

阻塞主线程.join():代表主线程要等待子线程执行完再继续执行
主线程中调用子线程的join(timeout)方法。
在start()之后调用。
主线程/进程等子线程结束后才会继续执行，如果不join，主线程/进程就会直接继续。

错误处理：
不能获取其他线程的错误，因此需要手动保存，手动在join函数处raise：
''import threading''

''class ExceptionThread(threading.Thread):''
	''def __init__(self, *args, **kwargs):''
		''threading.Thread.__init__(self, *args, **kwargs)''
		''self._exception = None''

	''def run(self):''
		''try:''
			''if self._target:''
				''self._target(*self._args, **self._kwargs)''
		''except Exception as e:''
			''self._exception = e''

	''def join(self, *args, **kwargs):''
		''super(ExceptionThread, self).join(*args, **kwargs)''
		''if self._exception:''
			''raise self._exception''
''def task1():''
	''# Some code that may raise an exception''
	''raise ValueError("An error occurred in task1")''

''def task2():''
	''# Some code that may raise an exception''
	''raise ValueError("An error occurred in task2")''

''thread1 = ExceptionThread(target=task1)''

''thread1.start()''
''try:''
	''thread1.join()''
''except Exception as e:''
	''print(f"Caught exception from thread1: {e}")''
