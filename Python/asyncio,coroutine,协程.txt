====== asyncio,coroutine,协程 ======
Created 星期三 21 六月 2023

https://docs.python.org/3/library/asyncio.html
https://www.bilibili.com/video/BV1oa411b7c9/

通过event loop进行调度，每个await会往loop里放一个task，并交回控制权。
loop只能等task主动交回控制权，所以如果有死循环就会卡死
本质上还是单线程


相比线程优点：
1. 轻量级：协程比线程更轻量级，创建和销毁协程的开销比线程小得多。
2. 更好的性能：由于协程在单个线程内运行，避免了线程切换的开销，因此在I/O密集型任务中性能更好。
3. 更好的控制：协程由程序员显式控制切换点，避免了线程的竞争和死锁问题。

使用场景：
高并发、IO密集（网络通讯）、资源受限、避免竞争和死锁
如爬虫、web服务器、实时数据处理（websocket）

await coroutine/task：block当前函数，交还控制权给event loop

控制同步：asyncio.Event()

**asyncio**
.run(coroutine object)：启动一个**新的**event loop，不断执行队列里的task，event loop获得控制权的时候会选一个task执行。全部执行完毕后结束loop，返回返回值。
！不在主线程，执行完run后需要配置一个新的loop：asyncio.set_event_loop(asyncio.new_event_loop())，而且这个loop要run起来，默认是没有运行的。
！原因：首次调用get_event_loop()时会自动建立一个loop，而调用.run会新建loop并覆盖asyncio默认的loop，运行完run后loop将close，此时如果不是主线程将无法自动产生新的loop
要想共用loop，需要底层手动新建loop，并设置set_event_loop

.gather(*coroutines) -> Awaitable: ==Promise.all，等所有coroutine都await，相当于同时做多个子任务

.create_task(coro) -> Task：把coro封装成task，注册到event loop，等待未来被运行，且不阻塞当前任务
.sleep(delaysec) -> Coroutine  # 设为0可以避免当前task长时间独占loop

.to_thread(func) -> Coroutine  # Asynchronously run function func in a separate thread. 用于IO bound任务，避免阻塞loop。可以结合.gather与其他task并行
https://docs.python.org/3.9/library/asyncio-task.html#id10

任务加入其他线程的loop：https://docs.python.org/3.9/library/asyncio-task.html#id11
.run_coroutine_threadsafe(coro, loop) -> Future  
	# Submit a coroutine to the given event loop. This function is meant to be called from a different OS thread than the one where the event loop is running.

不要滥用await，如果有多个await要善用create_task，这样各个task可以提前被运行，节约等待时间。

	''print("Task 2 finished")''
