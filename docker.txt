注意默认无法与宿主机通信
与宿主机通信
https://blog.csdn.net/DisMisPres/article/details/127279708
docker容器内默认情况下与主机进行了隔离，无法通过 127.0.0.1 或 localhost 访问到主机
解决方法：
1. docker run 添加 --network host 参数即可
2. ip addr show docker0 查询虚拟网卡的ip，发现为172.17.0.1，使用这个ip即可

配置镜像下载代理
1.=========
阿里云镜像加速 https://help.aliyun.com/zh/acr/user-guide/accelerate-the-pulls-of-docker-official-images
仅限阿里云用户，在支持公网访问的阿里云产品上使用该镜像加速能力
	加速器地址 https://nathqvfm.mirror.aliyuncs.com
	sudo mkdir -p /etc/docker
	sudo tee /etc/docker/daemon.json <<-'EOF'
	{
	  "registry-mirrors": ["https://nathqvfm.mirror.aliyuncs.com"]
	}
	EOF
	sudo systemctl daemon-reload
	sudo systemctl restart docker
2. =========
docker镜像仓库：
https://github.com/dongyubin/DockerHub
（理论上来说替换进上面的registry-mirrors即可

3. =========
配置代理
新版：https://docs.docker.com/engine/daemon/proxy/#httphttps-proxy
{
  "proxies": {
	"http-proxy": "http://127.0.0.1:10809",
	"https-proxy": "http://127.0.0.1:10809",
	"no-proxy": "localhost,127.0.0.1,.example.com,*.barryzzj.top"
  }
}
sudo systemctl restart docker

运行容器：
docker run -d -p 8000:8000 easychen/rsspush
交互式运行 -it
后台运行 -d
结束后自动删除容器 --rm

查看容器：
docker ps -a

查看容器log
docker logs <container_id>
-f 实时

恢复运行
docker start <container_id>

删除所有未使用的镜像、容器：
docker system prune -a

关闭容器：
docker stop <container_id>
删除容器:
docker rm <container_id>

查看镜像：
docker images
删除镜像：
docker rmi <image_name_or_id>
-f 强制删除
删除所有未使用的镜像：
docker image prune
