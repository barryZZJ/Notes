注意默认无法与宿主机通信
与宿主机通信
https://blog.csdn.net/DisMisPres/article/details/127279708
docker容器内默认情况下与主机进行了隔离，无法通过 127.0.0.1 或 localhost 访问到主机
解决方法：
1. docker run 添加 --network host 参数即可
2. ip addr show docker0 查询虚拟网卡的ip，发现为172.17.0.1，使用这个ip即可

1. =========================
sudo mkdir -p /etc/systemd/system/docker.service.d
sudo nano [[/etc/systemd/system/docker.service.d/proxy.conf]]

[Service]
Environment="HTTP_PROXY=http://127.0.0.1:10809/"
Environment="HTTPS_PROXY=http://127.0.0.1:10809/"
Environment="NO_PROXY=localhost,127.0.0.1,.example.com,yy.barryzzj.top"

sudo systemctl daemon-reload
sudo systemctl restart docker

运行容器：
docker run -d -p 8000:8000 easychen/rsspush
交互式运行 -it
后台运行 -d
结束后自动删除容器 --rm

查看容器：
docker ps -a

查看容器log
docker logs <container_id>
-f 实时

恢复运行
docker start <container_id>

删除所有未使用的镜像、容器：
docker system prune -a

关闭容器：
docker stop <container_id>
删除容器:
docker rm <container_id>

查看镜像：
docker images
删除镜像：
docker rmi <image_name_or_id>
-f 强制删除
删除所有未使用的镜像：
docker image prune
