====== desktop文件 ======
Created 星期日 23 七月 2023

设置快捷方式 (.desktop)
拷贝到 ''/usr/share/applications'' 或 ~/.local/share/applications 可以在库中显示

验证：
desktop-file-validate *.desktop
更新
sudo update-desktop-database

图标默认放在''/usr/share/icons 或'' ~/.local/share/icons后，Icon值可不加路径和后缀名
可以是png
https://blog.csdn.net/wuyujin1997/article/details/104655385

双击执行sh
https://blog.csdn.net/m0_65304012/article/details/127432802

[Desktop Entry]
Name=RDP
Exec=/home/barryoffice/rdp
Terminal=false
Type=Application
Icon=rdp
Categories=Game;

启动后Icon与点击的快捷方式不一致修复：
1. 确认窗口的 WM_CLASS：
xprop WM_CLASS
然后点击 xfreerdp 窗口。记下输出（例如 WM_CLASS(STRING) = "xfreerdp", "XFreeRDP"）。
2. 把 .desktop 的 StartupWMClass 设置为 xprop 输出中的其中一个字符串，例如 "StartupWMClass=xfreerdp"。保存后取消固定并重新固定该 launcher，或重启 GNOME Shell（Alt+F2 → r）或登出/登录。

合并多个快捷方式：actions
如果没有立即生效，重启 GNOME Shell（Alt+F2 → 输入 r → 回车）或登出/登录。
[Desktop Entry]
Name=RDP - twomon bar
Exec=xfreerdp /v:dorm.barryzzj.top:23389 /u:barry /p:zzj850462 /dynamic-resolution /home-drive /network:lan /multimon /monitors:0,1 /sound:sys:alsa +clipboard -heartbeat -toggle-fullscreen /log-level:OFF /floatbar:sticky:on,show:always 
Terminal=false
Type=Application
Icon=rdp
StartupWMClass=xfreerdp
Actions=Twomon;Onemon;
