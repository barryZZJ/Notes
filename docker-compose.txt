====== docker-compose ======
Created 星期五 26 五月 2023

同时管理一系列容器，读取目录下的docker-compose.yml配置文件
在有docker-compose.yml的目录下或用-f指定yml文件

yml配置：
代理：
''services:''
  ''web:''
	''environment:''
	  ''- HTTP_PROXY=http://127.0.0.1:10809''
	  ''- HTTPS_PROXY=http://127.0.0.1:10809''

端口映射：
本机端口:容器端口

默认容器网络与宿主机隔离，要想容器访问localhost的其他端口：
1. 使用host.docker.internal
2. 使用主机ip
3. 绑定端口（反向映射）
4. 修改网络模式为host（不推荐）：
在yml对应服务下添加：
''services:''
  ''web:''
	''network_mode: "host"''
此时端口映射不再有用

测试连通性：
nc -zv host.docker.internal 31200


start and restart all the services defined in docker-compose.yml
前台启动：
docker compose up
后台启动：
docker compose up -d --remove-orphans
查看日志（后台启动时）：
docker compose logs -f --timestamps

关闭：
docker compose stop
重启
docker compose start

卸载：（没有被mount的文件会被删除！）
docker compose down --remove-orphans
-v remove all volumes

确认是否删除成功：
docker ps -a
docker rm <container_id>

更新：
yml里image应以:latest结尾
docker compose pull

映射虚拟路径和实际路径：
volumnes:
	- ./host/path:/container/path
默认挂载在/var/lib/docker/volumes中
docker volume ls
docker volume inspect <id>

已经运行的容器新增挂载映射：
#1. 创建本地目录
mkdir -p ./mount/p
#2. 从容器中复制数据到本地
docker cp freshrss:/var/www/FreshRSS/p ./mount/
#3. 然后重新启动
docker-compose up -d
