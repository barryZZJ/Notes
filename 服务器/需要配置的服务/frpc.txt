====== frpc ======
Created 星期一 10 十月 2022

linux安装：
1. https://github.com/fatedier/frp
2. frpc放入/usr/bin
3. 注册为服务：
apt install systemd
sudo nano /etc/systemd/system/frpc.service
[Unit]
Description=Frp Client
After=network.target
Wants = network.target

[Service]
Type=simple
Restart=on-failure
RestartSec=60s
ExecStart=/usr/bin/frpc -c /home/barryoffice/frp/frpc_imserver.ini

[Install]
WantedBy=multi-user.target

4. sudo systemctl daemon-reload
systemctl start frpc
systemctl enable frpc

注册为windows服务，开机自启动：
https://gist.github.com/imyelo/b6c3d3d9383f7d5623f06a0c11052530
注册：
: 1. download [nssm](https://github.com/kirillkovalenko/nssm) and set its diretory into the global PATH environment value
: 2. replace the path below with where you placed frpc【管理员权限】
nssm install frpc "D:\Tools\frpc\frpc.exe" "-c D:\Tools\frpc\frpc.ini"
nssm set frpc DisplayName "frp client"
nssm start frpc

删除
nssm stop frpc
nssm remove frpc

加密：
如果公司内网防火墙对外网访问进行了流量识别与屏蔽，例如禁止了 ssh 协议等，通过设置 use_encryption = true，将 frpc 与 frps 之间的通信内容加密传输，将会有效防止流量被拦截。

开启加密后，数据包会变大。这里的加密是指 frpc 和 frps 之间的通信，一般是为了防止被防火墙或者网关过滤。

压缩：
如果传输的报文长度较长，通过设置 use_compression = true 对传输内容进行压缩，可以有效减小 frpc 与 frps 之间的网络流量，加快流量转发速度，但是会额外消耗一些 cpu 资源。

压缩针对的是每个包比较大的情况，如果都是小包，可能反而会有负面效果。
